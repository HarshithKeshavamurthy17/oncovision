# Contributing to OncoVision

Thank you for your interest in contributing to OncoVision! This document provides guidelines and instructions for contributing.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/oncovision.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes
6. Submit a pull request

## Code Style

- Follow PEP 8 style guidelines
- Use type hints where appropriate
- Write docstrings for all functions and classes
- Keep functions focused and modular

## Testing

Before submitting a pull request, please ensure:
- All existing tests pass
- New features have appropriate tests
- Code is properly formatted
- Documentation is updated

## Pull Request Process

1. Update README.md if needed
2. Update CHANGELOG.md if applicable
3. Ensure all tests pass
4. Request review from maintainers

## Questions?

Open an issue for any questions or concerns.
